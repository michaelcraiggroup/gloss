## Quick Reference

### macOS App Development

```bash
cd macos
swift build              # Build (SPM)
swift test               # Run tests (136 tests)
swift run                # Launch the app (SPM)

# Xcode project (for code signing, Quick Look extension, App Store)
xcodegen generate        # Generate Gloss.xcodeproj from project.yml
open Gloss.xcodeproj     # Open in Xcode в†’ set team в†’ Cmd+R
xcodebuild -project Gloss.xcodeproj -scheme Gloss build  # CLI build
xcodebuild -project Gloss.xcodeproj -scheme Gloss test   # CLI test
```

**Dual build modes:** SPM (`swift build/test`) for development, xcodegen-generated Xcode project for release (code signing, QL extension embedding, App Store). The `XCODE_BUILD` compilation condition distinguishes them вЂ” see `GlossApp.swift:setAppIcon()`.

### Extension Development

## Architecture

### macOS App

The macOS app uses **GlossKit**, a shared library also used by the Quick Look extension:

1. `GlossKit.MarkdownRenderer.render()` вЂ” parses markdown with `swift-markdown`, converts to HTML via `HTMLFormatter.format()`, wraps in themed document
2. HTML post-processing: heading IDs (slug generation), wiki-link resolution, anchor links
3. HTML includes: CSS theme (custom properties), highlight.js, copy buttons (JS), heading anchors (JS), vim-style keyboard navigation (JS)
4. Renders in `WKWebView` via `NSViewRepresentable` (first responder for keyboard nav)
5. `isDark: Bool?` вЂ” explicit class for app, `nil` for Quick Look (uses `prefers-color-scheme`)
6. `fontSize: Int` вЂ” injectable via CSS variable override (`--font-size`)
7. `resolveWikiLink` вЂ” optional closure for `[[wiki-link]]` resolution
8. Inspector sidebar: `extractHeadings()` walks markdown AST, `extractFrontmatter()` parses YAML via Yams
9. StoreKit 2: `StoreManager` manages IAP, `PaywallView` for purchase UI, `PaidFeature` enum for gating

Theme CSS uses CSS custom properties with `prefers-color-scheme` and explicit `html.dark`/`html.light` class overrides for app-controlled appearance.

### Editor Mode (Phase 7)

DocumentView supports read/edit toggle (Cmd+Shift+E):

1. **Read mode** вЂ” current rendered HTML via `WebView` (unchanged)
2. **Edit mode** вЂ” CodeMirror 6 live preview via `EditorWebView`
3. CM6 loaded from `editor.html` bundled resource, imports from esm.sh CDN
4. **Live preview** вЂ” Obsidian-style inline rendering: hides markdown syntax (`#`, `**`, `*`, etc.) on non-focused lines, renders headings at proper sizes, links as styled text, checkboxes as interactive widgets
5. **Swift в†” JS bridge**: `WKScriptMessageHandler` receives `ready`, `save`, `dirty` messages; Swift calls `glossEditor.setContent()` / `getContent()` / `markClean()`
6. **Auto-save**: on mode switch (edit в†’ read) and Cmd+S
7. **File operations**: `FileTreeModel.createFile()`, `renameItem()`, `deleteItem()` with sidebar context menus

### Link Index (Phase 8)

Persistent link index for knowledge management:

1. **GRDB.swift** dependency (Gloss app target only, not GlossKit)
2. **SQLite schema** at `.gloss/index.sqlite` in vault root вЂ” `files`, `links`, `tags` tables
3. **`LinkDatabase`** вЂ” `Sendable` struct wrapping `DatabaseQueue`, all CRUD + queries
4. **`LinkIndex`** вЂ” `@Observable @MainActor` orchestrator, builds full index on folder open
5. **Typed wiki-links**: `[[target::supports]]`, `[[target::type|display]]` вЂ” 8 link types
6. **`extractLinks()`** / **`extractTags()`** вЂ” GlossKit public APIs for parsing wiki-links and frontmatter tags
7. **Backlinks** in inspector: grouped by `LinkType` with `DisclosureGroup`, click navigates to source file
8. **Incremental updates**: re-indexes on editor save, file create/rename/delete
9. **Path standardization**: uses `URL.standardizedFileURL.path` to avoid symlink mismatches

### Feature Walkthroughs (Phase 8.6)

Hybrid walkthrough system integrating Rabble Guide SDK (web) + native SwiftUI overlays:

1. **GlossGuideService** вЂ” `@Observable @MainActor` coordinator, sequences steps one at a time
2. **Two renderers**: Rabble Guide SDK for WKWebView content (spotlight, content steps), `NativeSpotlightOverlay` for SwiftUI views
3. **SpotlightPreference system** вЂ” `PreferenceKey` collects `(SpotlightTarget, CGRect)` pairs from annotated views
4. **GuideInjector** вЂ” injects IIFE bundle into rendered HTML via `</body>` replacement
5. **WebView bridge** вЂ” `WKScriptMessageHandler` for `glossGuide` channel, notification-based communication
6. **Guide definitions** вЂ” static `WalkthroughGuide` instances: "Getting Started" (7 steps) + "What's New: Tags" (3 steps)
7. **Trigger UX** вЂ” Help menu items, `AppStorage` for onboarding/version tracking

### Reading Mode

The extension intercepts markdown file opens and:

1. Triggers the built-in markdown preview
2. Closes the source editor tab
3. Creates a "read-only by default" experience

## Privacy

No telemetry, no analytics. CDN resources (highlight.js, mermaid.js, KaTeX) loaded for rendering only when needed. No data sent anywhere.
